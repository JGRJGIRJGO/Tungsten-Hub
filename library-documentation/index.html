Lyra UI Library V4

A compact Roblox interface library with responsive windows, mouse and touch input, runtime themes, optional key verification, notifications, and Lucide icon support. V4 keeps the original positional API while adding table-style configuration.

API styleTable and positional
Window inputDrag, resize, toggle
TypographyMontserrat interface
Icon systemLucide and asset IDs
Existing scripts using calls such as CreateButton("Run", callback) remain compatible. Table configuration is recommended for new code because it supports icons and named options.
Installation

Load the library

The library source is LyraV2.lua. The Universal script is a consumer of the library, not the UI module itself.

HTTP loader
local Lyra = loadstring(game:HttpGet(
    "https://raw.githubusercontent.com/JGRJGIRJGO/Tungsten-Hub/main/LyraV2.lua"
))()
ModuleScript
local Lyra = require(path.To.LyraV2)
The HTTP form requires an environment that exposes both game:HttpGet and loadstring. Use a ModuleScript in environments where those APIs are unavailable.
Window API

Create and manage a window

Lyra:CreateWindow() creates the ScreenGui, sidebar, top bar, content host, resize grip, mobile toggle, and optional key gate.

Table configuration
local Window = Lyra:CreateWindow({
    Name = "Lyra Tools",
    Subtitle = "Utilities",
    Icon = "command",
    Size = Vector2.new(760, 500),
    MaxWidth = 1200,
    MaxHeight = 800,
})

Tab API

Build the sidebar navigation

Tabs accept a simple name or a table containing a name and icon. The first tab selects itself automatically.

Recommended
local MainTab = Window:CreateTab({
    Name = "Main",
    Icon = "home",
})

local SettingsTab = Window:CreateTab({
    Name = "Settings",
    Icon = "settings",
})
Legacy positional form
local MainTab = Window:CreateTab("Main")
MethodDescription
Tab:Select()Selects the tab, updates the top-bar title, and shows its page.
Tab:SetName(name)Updates the sidebar label, monogram fallback, and active-page title.
Tab:SetIcon(icon)Replaces the current icon. An unresolved value falls back to the tab monogram.
Control API

Add controls to a tab

Every control supports the original positional call and a table configuration. The table form exposes icons, named defaults, and additional options without changing argument order.

Label / Section

Tab:CreateLabel(config)

Creates a section heading. CreateSection is an alias of CreateLabel.

Label
local Status = MainTab:CreateLabel({
    Text = "Ready",
    Icon = "info",
})

Status:SetText("Recording")
Text | NameIconUpdateText(value)SetText(value)

Button

Tab:CreateButton(config)

Runs a command callback when clicked. Disabled buttons ignore clicks.

Button
local RunButton = MainTab:CreateButton({
    Name = "Run task",
    Icon = "play",
    Callback = function()
        print("Running")
    end,
})

RunButton:SetDisabled(false)
Name | TextIconCallbackDisabledUpdateButtonText(value)SetDisabled(bool)

Toggle

Tab:CreateToggle(config)

Stores a boolean state. Its callback receives the initial state once during creation and then receives every change.

Toggle
local AutoSave = MainTab:CreateToggle({
    Name = "Auto save",
    Icon = "save",
    CurrentValue = true,
    Callback = function(enabled)
        print("Auto save:", enabled)
    end,
})

AutoSave:SetValue(false)
print(AutoSave:GetValue())
Name | TextIconCurrentValue | DefaultCallbackSetToggle(bool)SetValue(bool)GetValue()

Slider

Tab:CreateSlider(config)

Produces a clamped number. Increment controls rounding and decimal display.

Slider
local Volume = MainTab:CreateSlider({
    Name = "Volume",
    Icon = "volume-2",
    Range = { 0, 100 },
    Increment = 5,
    CurrentValue = 50,
    Callback = function(value)
        print("Volume:", value)
    end,

Volume:SetValue(75)

Dropdown

Tab:CreateDropdown(config)

Shows a centered Montserrat option selector in an overlay popup. CurrentOption may be a value or a one-item table.

Dropdown
local Mode = MainTab:CreateDropdown({
    Name = "Mode",
    Icon = "list-filter",
    Options = { "Record", "Replay", "Idle" },
    CurrentOption = "Record",
    Callback = function(value)
        print("Mode:", value)
    end,
})

Mode:Refresh({ "Record", "Replay" })
Mode:Select("Replay")
Name | TextIconOptions | ListCurrentOption | DefaultCallbackSelect(value)Refresh(options)GetValue()

Textbox

Tab:CreateTextbox(config)

Collects centered text input. The callback runs when focus is lost and receives text plus enterPressed.

Textbox
local Tag = MainTab:CreateTextbox({
    Name = "Run tag",
    Icon = "text-cursor-input",
    PlaceholderText = "Type a tag",
    CurrentValue = "default",
    Callback = function(text, enterPressed)
        if enterPressed then
            print("Submitted:", text)
        end
    end,
})

Tag:SetText("nightly")
Name | TextIconPlaceholderText | PlaceholderCurrentValueCallback(text, enterPressed)SetText(value)GetText()

Legacy positional calls

Backward-compatible forms
Tab:CreateLabel("Section")
Tab:CreateButton("Run", callback)
Tab:CreateToggle("Enabled", false, callback)
Tab:CreateSlider("Amount", 0, 100, 50, callback)
Tab:CreateDropdown("Mode", { "A", "B" }, "A", callback)
Tab:CreateTextbox("Tag", "Type here", callback)
Returned handles accept both dot and colon calls. For example, Status.SetText("Ready") and Status:SetText("Ready") are both supported.
Notifications

Show transient status

Notifications stack from the bottom-right, include a timer bar, and can be dismissed manually. A duration of zero keeps the notification open.

Table configuration
local Notice = Window:Notify({
    Title = "Recorder armed",
    Content = "Waiting for an elevator.",
    Duration = 4,
    Icon = "radio-tower",
})

-- Notice:Dismiss()
Positional form
Window:Notify("Saved", "Configuration updated.", 3)
Icon API

Use Lucide names or Roblox assets

Lyra includes the icons used by its own interface. Register tijnepema/lucide-roblox to resolve the package's complete icon catalog by name.

Provider setup
local Lucide = require(path.To.Lucide)

Lyra:SetIconProvider(Lucide)

local Window = Lyra:CreateWindow({
    Name = "Icon demo",
    Icon = "command",
    Lucide = Lucide,
})

local Tab = Window:CreateTab({
    Name = "Dashboard",
    Icon = "layout-dashboard",
})

Optional gate

Validate a local key

The key gate compares user input with a configured string, one of several strings, or a validation function. It does not collect an HWID or perform remote verification by itself.

KeySettings
local expectedKey = "Lyra_" .. Lyra.SHA256("your-payload"):upper()

local Window = Lyra:CreateWindow({
    Name = "Protected tools",
    Subtitle = "Verification",
    KeySettings = {
        Title = "Key verification",
        Subtitle = "Protected tools",
        Note = "Enter your key to continue.",
        Icon = "key-round",
        SaveKey = true,
        Key = expectedKey,
        Url = "https://example.com/get-key",
    },
})
PropertyTypeDescription
TitlestringGate heading.
SubtitlestringSmaller heading below the title.
NotestringInstructions shown above the input.
IconIconValueGate heading icon.
Keystring | table | functionExpected value, accepted value list, or validator returning true.
SaveKeybooleanUses readfile/writefile when available to cache a successfully validated key.
UrlstringCopied to the clipboard when Get key is pressed.
The cache file name is derived from the window title and has no built-in expiration timer. A saved value is reused only when it still passes the current Key validator.
Full example

Put the APIs together

This example uses table configuration, built-in icons, returned handles, themes, a notification, and the current loader URL.

Complete script
local Lyra = loadstring(game:HttpGet(
    "https://raw.githubusercontent.com/JGRJGIRJGO/Tungsten-Hub/main/LyraV2.lua"
))()

local Window = Lyra:CreateWindow({
    Name = "Lyra Demo",
    Subtitle = "V4 controls",
    Icon = "command",
    Size = Vector2.new(780, 520),
})

local Main = Window:CreateTab({ Name = "Main", Icon = "home" })
local Settings = Window:CreateTab({ Name = "Settings", Icon = "settings" })

local Status = Main:CreateLabel({ Text = "Ready", Icon = "info" })

Main:CreateToggle({
    Name = "Auto save",
    Icon = "save",
    CurrentValue = false,
    Callback = function(enabled)
        Status:SetText(enabled and "Auto save enabled" or "Auto save disabled")
    end,
})

Main:CreateSlider({
    Name = "Volume",
    Icon = "sliders-horizontal",
    Range = { 0, 100 },
    Increment = 5,
    CurrentValue = 50,
    Callback = function(value)
        print("Volume:", value)
    end,
})

Main:CreateDropdown({
    Name = "Mode",
    Icon = "list-filter",
    Options = { "Record", "Replay", "Idle" },
    CurrentOption = "Idle",
    Callback = function(value)
        Status:SetText("Mode: " .. value)
    end,
})

Main:CreateTextbox({
    Name = "Run tag",
    Icon = "text-cursor-input",
    PlaceholderText = "Type a tag",
    Callback = function(text, enterPressed)
        if enterPressed then
            Status:SetText("Tag: " .. text)
        end
    end,
})

Main:CreateButton({
    Name = "Show notification",
    Icon = "bell",
    Callback = function()
        Window:Notify({
            Title = "Lyra Demo",
            Content = "The current V4 API is running.",
            Duration = 4,
            Icon = "circle-check",
        })
    end,
})

Settings:CreateDropdown({
    Name = "Theme",
    Icon = "settings",
    Options = { "Lyra", "Nebula", "BloodMoon", "Emerald", "Midnight" },
    CurrentOption = "Lyra",
    Callback = function(name)
        Window:SetTheme(Lyra.Themes[name])
    end,
})

Settings:CreateButton({
    Name = "Close interface",
    Icon = "x",
    Callback = function()
        Window:Destroy()
    end,
})

Window:Notify("Loaded", "Lyra UI Library V4 is ready.", 4)
